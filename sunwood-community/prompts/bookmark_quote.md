ブックマーク通知向けに、sunwood-community skill で解説投稿を作成してください。

作業フロー:
1. まず、指定された保存先に調査レポート Markdown を作成してください。
2. 調査レポートには、要約、キーワード、技術要点、関連する公式情報、時系列の見解、投稿方針を整理してください。
3. 解説本文は、その調査レポートを元に組み立ててください。
4. 図解画像は、その調査レポートを元に内容を抽出して作成してください。元ポスト原文をそのまま図解に流し込まないでください。
5. 図解は日本語で作成してください。
6. 投稿直前に、`skills/sunwood-community/scripts/post_thread.py` 用の payload JSON を作成してください。
7. 本体投稿、画像添付、複数リプライは、その payload を使って `post_thread.py --payload-file ...` を1回だけ実行して投稿してください。

共通投稿ルール:
- `prompts/shared_posting_rules.md` を適用してください。

共通の解説要件:
- `prompts/shared_explainer_requirements.md` を適用してください。

共通の結果要件:
- `prompts/shared_result_requirements.md` を適用してください。

ブックマーク固有ルール:
- ブックマークした元ツイートURLは、必ずその自分の投稿へのリプライに貼ってください。
- 添付リンクがある場合も、本文ではなくその自分の投稿へのリプライに貼ってください。
- 元ツイートURLのリプライには、その投稿の論点や注目ポイントを1から2文で添えてください。
- 添付リンクURLのリプライには、そのリンク先の内容と読む目的を1から2文で添えてください。
- 直近ログと重複していたら無視してください。
- 時系列の流れや前後関係がある場合は、その見解も入れてください。
- 調査レポート保存先が指定されている場合は、必ずその Markdown を先に作成してください。
